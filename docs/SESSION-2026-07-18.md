# Session 2026-07-18 â€” ZIWO Admin Dashboard & 500 Error Fixes

## Problem Summary

All pages returning 500 errors. `/ziwo/dashboard` and `/ziwo/statistics` needed real API data from `nayatel-api.aswat.co` proxy, with Alpine.js dashboard consuming admin API endpoints.

## Root Causes Found & Fixed

### 1. PHP ParseError â€” DashboardController (first session)
- `DashboardController.php` had `try {` without `catch` after bad edit
- ParseError cascaded to RouteNotFoundException on every page

### 2. PHP ParseError â€” ZiwoClient.php
- Admin mock endpoint `if` blocks replaced `users/login` mock's `if (str_contains(...))` line, leaving orphan `return [...]` outside any function
- PHP refused to compile entire app â†’ every route returned 500

### 3. Route Name Mismatch
- `routes/web.php` registered `ziwo.dashboard` / `ziwo.statistics` as `admin.ziwo.dashboard` / `admin.ziwo.statistics`
- Layout `app.blade.php` used `route('ziwo.dashboard')` â†’ `Route [ziwo.dashboard] not defined`
- **Fix:** Changed route names to `ziwo.dashboard` / `ziwo.statistics` in `web.php`

### 4. Missing View Variables
- `TelephonyAdminController::dashboard()` and `statistics()` returned view without `$analytics`, `$callLogs`, `$webhookLogs`
- View tried `$callLogs->links()` â†’ `Undefined variable` error
- **Fix:** Pass required vars from `$this->service->getDashboardAnalytics()` + empty paginator

### 5. Alpine Component Not Registering on SPA Nav
- Blade used `document.addEventListener('alpine:init', () => Alpine.data('telephonyDashboard', ...))`
- bento-bridge SPA navigation re-executes inline scripts â†’ `alpine:init` already fired on first load â†’ callback never runs â†’ `telephonyDashboard` undefined â†’ all Alpine expression errors (`liveStats`, `refreshing`, `onlineAgentsCount`)
- **Fix:** Wrap in `if (window.Alpine) { Alpine.data(...) } else { document.addEventListener('alpine:init', ...) }`

### 6. Module Route Override
- `Modules/ZiwoAdmin/routes/web.php` registered same route names (`ziwo.dashboard`, `ziwo.statistics`) â†’ loaded after `routes/web.php` â†’ overwrote our routes
- `ZiwoAdminController::statistics()` checked `session('ziwo_admin_token')` â†’ missing â†’ redirect login â†’ login redirects dashboard â†’ user always lands on dashboard regardless of URL
- **Fix:** Removed `/dashboard` and `/statistics` from module routes, kept `/export/{format}` and `/login`
